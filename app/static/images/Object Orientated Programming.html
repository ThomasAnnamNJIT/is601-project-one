Object Orientated Programming (OOP)

Object Orientated Programming also known as OOP is a software design model that allows developers and their teams to write code around modeling objects, which can illustrated through classes and methods. Classes allow developers to build blueprints around the object to be modeled (i.e. abstractions) which are defined via fields, instance methods, static methods, etc. Meanwhile, methods are actions/behaviors that are defined for a given class. Methods are used to define instructions to perform a repetitive operation in order to fetch a result, which can be generalized in one way or form. Now that we are familiar with OOP and some cornerstones that OOP entails (i.e. classes and methods), we will be discussing the following terms:

  • Abstraction
  • Encapsulation
  • Inheritance
  • Polymorphism

Docker Commands

docker build -t [image_name] [path]
This allows you to build an image from the location of your Dockerfile and allows you to tag it with a name.
docker kill [container_id]
This allows you to destroy/remove a container without having to stop it beforehand
docker system prune
This command will destroy any containers that are stopped containers or remove any dangling images, which include those that are untagged.
docker run [image_id]
This command allows you to run a container after you have build an image from the docker build command; however, if you simply run this command without having built an image, it will build an image first, and then run the container for you automatically!
docker ps
This will list all of your running/stopped/exited containers
docker compose up
This commands allows you to bring multiple containers up at once within a service such as your application container and a db container, and then connecting your application to the DB. In addition, this will also create a network for you in order for your containers to communicate with each other. This is commonly defined within a docker.compose.yml file.
docker compose down
This commands will destroy all the containers associated with the service, along with any networks/volumes created

Project Setup

Bring Up Containers
To bring up the project please run docker compose up in the command line from the root of the project directory where your docker.compose.yml file is located, this will build/run the image associated with the Dockerfile and create a network configuration. Within the docker.compose.yml file, this application container maps from "80:5000", this mean you'll be able to access it on port 80, which would then map to port 5000 within the container.
Access the Container
To access the container, run docker exec -it [container_id] sh, this will essentially give you an interactive shell for the container
