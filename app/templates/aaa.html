Testing

Testing is one of the most important benefits within the software development lifecycle. This is because it allows development teams/developers to find any bugs/defects within their code before going into production, ensures quality code is written, and, lastly determines whether both functional and business requirements are being met for the software being rolled out. A more precise definition of testing can be in the form of software testing, which is defined as "...the process of evaluating and verifying that a software product or application does what it is supposed to do" (https://www.ibm.com/topics/software-testing).
The two most common software testing are integration/unit testing. Unit testing is testing a particular piece of code (i.e a method), which can be thought of as the subject under test. Typically, with unit testing, developers mock out any external dependencies such as database operations or 3rd party API calls. Integration testing involves testing all the entire software application, which can be from a UI to the backend or an API to call the backend (via Postman or curl). Now that we are familiar with what testing is, the different types of tests, and why they are important in the role of software development, we will now discuss a pattern for writing quality tests, which is known as AAA testing.
