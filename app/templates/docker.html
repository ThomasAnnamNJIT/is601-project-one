Docker

Docker is a containerization technology/platform for creating and managing containers. The core building blocks for Docker include containers and images. Docker provides an entire ecosystem for building images and containers some of which include:

  • Docker Machine
  • Docker Hub
  • Docker CLI
  • Docker Compose

Containers and Images

Images are the blueprints for building a container which are commonly defined via a Dockerfile or a docker-compose.yml. Within a Dockerfile, developers providers instructions to build a container such as:

  • FROM
  • RUN
  • CMD
  • ENTRYPOINT
In addition, images contain all of our code and all the dependencies that we want to run before deploying a container. This can include installing dependencies from a package.json via the RUN command.

Containers are the runtime version of the image, or the actual running unit of code that we built from the image with the Dockerfile. A container allows us to output the same application and same behavior, regardless of where it is executed, in other words, we're essentially "packaging" our entire machine. This allows for distribution of an application to be efficient.

Even though you can pull base images from a custom registry such as ECR, most base images come from DockerHub, which is essentially the app store/repository for all base images).
