Docker

Docker is a containerization technology/platform for creating and managing containers. The core building blocks for Docker include containers and images. Docker provides an entire ecosystem for building images and containers some of which include:

  • Docker Machine
  • Docker Hub
  • Docker CLI
  • Docker Compose

Containers and Images

Images are the blueprints for building a container which are commonly defined via a Dockerfile or a docker-compose.yml. Within a Dockerfile, developers providers instructions to build a container such as:

  • FROM
  • RUN
  • CMD
  • ENTRYPOINT
In addition, images contain all of our code and all the dependencies that we want to run before deploying a container. This can include installing dependencies from a package.json via the RUN command.

Containers are the runtime version of the image, or the actual running unit of code that we built from the image with the Dockerfile. A container allows us to output the same application and same behavior, regardless of where it is executed, in other words, we're essentially "packaging" our entire machine. This allows for distribution of an application to be efficient.

Even though you can pull base images from a custom registry such as ECR, most base images come from DockerHub, which is essentially the app store/repository for all base images). Click here to view this project's Docker image on DockerHub

Docker Commands

docker build -t [image_name] [path]
This allows you to build an image from the location of your Dockerfile and allows you to tag it with a name.
docker kill [container_id]
This allows you to destroy/remove a container without having to stop it beforehand
docker system prune
This command will destroy any containers that are stopped containers or remove any dangling images, which include those that are untagged.
docker run [image_id]
This command allows you to run a container after you have build an image from the docker build command; however, if you simply run this command without having built an image, it will build an image first, and then run the container for you automatically!
docker ps
This will list all of your running/stopped/existed containers for you.
docker compose up
This commands allows you to bring multiple containers up at once within a service such as your application container and a db container, and then connecting your application to the DB. In addition, this will also create a network for w This is commonly defined within a docker.compose.yml file.
docker compose down
This commands will destroy all the containers associated with the service, along with any networks/volumes created

Project Setup

Bring Up Containers
To bring up the project please run docker compose up command from the root of the project directory where your docker.compose.yml file is located, this will build/run the image associated with the Dockerfile and create a network configuration. Within the docker.compose.yml file, this application container maps from "80:5000", this mean you'll be able to access it on port 80, which would then map to port 5000 within the container.
Access the Container
To access the container, run docker exec -it [container_id], this will essentially give you an interactive shell for the container
