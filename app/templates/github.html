GitHub

GitHub is a platform that allows developers to host/collaborate their codebase on the Web. One of the main features GitHub provides is version control, which keeps track of all the changes, a developer has made within a codebase. This can be shown via commits, issues, tags, or pull requests.

GitHub Tools

GitHub provides many services to developers, some of which include:

  • GitHub CLI
  • GitHub Actions
  • GitHub Desktop
  • Git Bash (For Windows)

GitHub Commands

Repository
A repository in GitHub is essentially a directory where you host all your team's folders and files for the codebase. To initialize a GitHub repository, this can be done via the GitHub UI, and then can be cloned locally or simply via git init, which will mark the current directory as a GitHub repository.
Commit
A GitHub commit refers to a set of changes made to a given codebase at any point in time. Once a commit is made to GitHub, each commit will have a message and a unique git commit SHA. To make a git commit, one would need to execute the following series of commands:
  • git add .
    • This will add of your files that are currently "not staged for commit" into your commit.
  • git commit -m 'commit message'
    • This will give your Git commit a message before it's pushed into a branch
  • git push origin branch_name
    • This will push all your changes with the commit message to the branch_name
Branches
A GitHub branch allows a developer to independently develop new features or fix issues, independently off the main branch or any other branch within a given repository. Most branches in GitHub are branched out of a main branch which include the names such as main, master, or develop. To create a branch, one would need to issue the following command: git checkout -b branch_name, this will make a new branch of the base branch if it doesn't already exist.
Tags
A GitHub tag is reference to a specific point for a repository, which typically marks when you want to release this code into production or package your code as version for a library, such as to maven central repository or PIP.
Merging
A GitHub merge allows you to take one branch and put it another branch. This is typically done via a Pull Request, which is an official way to say you want to merge your current branch into another branch such as master, and if it has been reviewed by the team of developers.
