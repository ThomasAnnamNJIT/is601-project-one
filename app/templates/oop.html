OOP

Object Orientated Programming is a software technique that allows software development teams/developers to model their functional and business requirements (i.e code) based on objects is one of the most important benefits within the development cycle, which are done via classes. These are done via classes and methods. Classes allow developers to build blueprints around the object(s) to be modeled (i.e. abstractions) which are defined via fields, instance methods, static methods, etc. Meanwhile, methods are actions/behaviors that are defined for a given class. Methods are used to define instructions to perform a repetitive operation in order to fetch a result, which can be generalized in one way or form. Now that we are familiar with OOP and some cornerstones that OOP entails (i.e. classes and methods), we will be discussing the following terms:

  • Abstraction
  • Encapsulation
  • Inheritance
  • Polymorphism

Encapsulation
Encapsulation means only methods in the class with the variables can refer to the instance variable. In other words, only certain fields of a given classes are able to called from other parts of your program. The three scopes that Python has includes public (this is the default), protected, and private. Public allows a field/method to be accessed anywhere in your program. Protected allows access for only the class itself as well as any subclasses. Lastly, private access only allows for the class itself to access it's methods.
