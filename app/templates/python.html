Python Flask

Flask is a popular Python MVC framework that allows developers to web applications. Flask is considered a light-weight framework given how flexible it is in terms of what libraries you're able to choose from or not. To develop a web app for Flask, developers can use both blueprints and templates. Templates are essentially all of your HTML files, which are based of the Jinja2 templating engine. Blueprints are where you define all your routes and standard REST APIs (i.e. GET, POST, DELETE, PUT) operation, this is where you will perform a business logic, this can vary in terms of simply outputting a string, an HTML page, or forwarding into your service layer.

Project Layout

.github/workflows
This is where you define your GitHub workflows for your development/prod environment using GitHub Actions.
app/templates
This is where of all your HTML files will be placed, and that will be rendered when you add your template_folder to your BluePrint configuration.
app/static
This is where of all your static content will go such as css, js, and images.
app/__init__.py
Initializes that this directory is a python package, and this file initializes a flask application and registers of all your blueprints (i.e. API).
app/run.py
Runs the application
app/content/__init__.py
This is a python package which currently holds the BluePrint configuration for content pages
tests
This is a python package where all of your unit tests will be placed under
tests/content_pages_test.py
This is a unit test which tests all the routes that are defined in within the content pages (BluePrint)
tests/conftest.py
This setups any pytext fixtures such as a client for http applications
.gitignore
This is a special file for a GitHub repository which will essentially ignore any file that is listed under this and will never be part of the commit.
docker-compose.yml
This will bring up all the services associated with your project. In this situation, it will build and run the image that is defined from your Dockerfile and also define a port mapping from port 80 on the host to port 5000 within the container. In addition, this will start the application by running "flask run"
Dockerfile
This serves as a blueprint to build a docker image for your flask application.
heroku.yml
Configuration file that will allow us to deploy our container to Heroku
pytest.ini
Configuration file that tells pytest at which directory our python tests will be located
README.md
This provides a high-level overview of the project with all of its components associated with it
requirements.txt
This file is where you define all your python packages that you want as dependencies
