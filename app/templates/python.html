Python Flask

Flask is a popular Python MVC framework that allows developers to build web applications. Flask is considered a light-weight framework given how flexible it is in terms of what libraries you're able to choose from or not. To develop a web app for Flask, developers can use both blueprints and templates. Templates are essentially all of your HTML files, which are based of the Jinja2 templating engine. Blueprints are where you define all your routes and standard REST APIs (i.e. GET, POST, DELETE, PUT) operation, this is where you will perform a business logic, this can vary in terms of simply outputting a string, an HTML page, or forwarding into your service layer.

PyTest

PyTest is a testing framework in Python that allows you to write unit tests. I was able to create all my content pages using Jinja2, which is templating engine in Python to render html content. I first setup a Flask Blueprint as defined within this directory app/content/__init__.py, and created routes for all the html pages and the locations to find the templates and static folder (also configured in the BluePrint) for render_templates. To get started with pytest, you must have a pytest.ini file, which allows you to give the path to where your tests are located via testpaths.

Project Layout

.github/workflows
This is where you define your GitHub workflows for any environment using GitHub Actions.
app/templates
This is where of all your HTML files will be placed, which will then be rendered when you add your template_folder to your BluePrint configuration.
app/static
This is where of all your static content will go which includes css, js, and images.
app/__init__.py
Initializes this directory a python package, and this file initializes a flask application and registers of all your associated blueprints (i.e. API).
app/run.py
Starts/runs the flask application.
app/content/__init__.py
This is a python package which currently holds the BluePrint configuration for the content pages (i.e. your controller/API).
tests
This is a python package where all of your unit tests should be placed.
tests/content_pages_test.py
This is a python file which contains your unit tests for your code. In this example, it will test all the routes that are defined within the content pages (BluePrint).
tests/conftest.py
This setups any pytext fixtures such as a client for http applications.
.gitignore
This is a special file for a GitHub repository which will essentially ignore any file that is listed under this and will never be part of a commit.
docker-compose.yml
Orchestration file that contains the configuration to develop locally, it overrides the Dockerfile to run the flask development server instead of running the gunicorn server that is used for hosting on Heroku.
Dockerfile
File used to create an image to run in a container that runs the program.
heroku.yml
Configuration file that will allow us to deploy our container to Heroku.
pytest.ini
Configuration file that tells pytest at which directory our python tests will be located at.
README.md
This provides a high-level overview of the project with all of its components associated with it.
requirements.txt
This file is where you define all your python packages that you want as dependencies for your project.
