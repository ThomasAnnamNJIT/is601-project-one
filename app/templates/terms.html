Class
A class is a blueprint that allow developers to build on abstractions they want to model for an object, to meet both business and functional requirements for their application.

Principles of OOP

Encapsulation
Encapsulation means only methods in the class with the variables can refer to the instance variable. In other words, only certain fields of a given classes are able to called from other parts of your program. The three scopes that Python has includes public (this is the default), protected, and private.
Inheritance
Inheritance allows one class to inherit attributes and methods from one class to another.
Polymorphism
Polymorphism allows an object to take on many forms, this means for a given Python method, we can base in the base class as an argument, and when we use the base class and call a method, it will have a different behavior. This is shown in the Calculator example, with the perform_operations method between Addition, Subtraction, or Multiplication.
Abstraction
Classes are blueprints around the object(s) to be modeled. Abstraction is a way to provide common interfaces of methods and APIs, which are then implemented by their subclasses.
